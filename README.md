# UTB/OS

UTB/OS — это маленькая, но **реальная низкоуровневая система**.
Она не запускается как обычная программа в Windows или Linux. Вместо этого она грузится как отдельный `x86`-образ через BIOS-совместимый эмулятор.

Проще говоря:

- это **не** сайт и не приложение на HTML/CSS;
- это **не** программа поверх Windows;
- это маленькая загрузочная система со своим boot sector и своим ядром.

## Что уже есть в проекте

В проекте уже реализованы:

- загрузочный сектор `boot.asm`;
- ядро `kernel.asm`;
- вывод прямо в VGA-память `0xB8000`;
- чтение клавиш через BIOS;
- текстовый интерфейс `80x25`;
- три встроенные программы:
  - `System Shell`
  - `Monitor`
  - `Profile Inspector`
- более красивый интерфейс с панелями и подсветкой активного окна;
- help-оверлей, который можно открыть прямо из ядра;
- дополнительные BIOS-метрики: память, video mode, equipment word, RTC-время;
- более безопасная загрузка: bootloader теперь читает ядро с повторными попытками.

## Как это работает очень коротко

При запуске происходит так:

1. BIOS читает первый сектор диска.
2. Этот сектор — `boot.asm`.
3. `boot.asm` загружает `kernel.asm` в память.
4. Ядро рисует интерфейс, показывает BIOS-метрики и ждёт нажатия клавиш.

# Сборка и запуск простыми словами

Ниже — самая простая инструкция без лишней теории.

## Что нужно установить

Нужны 3 инструмента:

1. `nasm` — assembler, который собирает `.asm` файлы;
2. `make` — чтобы запускать сборку одной командой;
3. `qemu-system-i386` — чтобы запустить систему в эмуляторе.

---

## Windows 10: самый простой способ

Если у тебя Windows 10, самый удобный вариант — **MSYS2**.

### Шаг 1. Установи MSYS2

Скачай и установи MSYS2.
После установки открой терминал **MSYS2 UCRT64**.

### Шаг 2. Установи нужные пакеты

В терминале MSYS2 выполни:

```bash
pacman -Syu
```

Если терминал попросит перезапуск — закрой его и открой снова, потом выполни:

```bash
pacman -S make nasm mingw-w64-ucrt-x86_64-qemu
```

После этого у тебя будут:

- `make`
- `nasm`
- `qemu-system-i386`

### Шаг 3. Проверь, что всё установилось

Выполни:

```bash
make --version
nasm -v
qemu-system-i386 --version
```

Если команды выводят версии, значит всё готово.

> Если тебе нужен только `make`, его всё равно удобнее ставить вместе с MSYS2, потому что там же легко поставить `nasm` и `qemu`.

---

## Linux: коротко

Пример для Debian/Ubuntu:

```bash
sudo apt update
sudo apt install make nasm qemu-system-x86
```

# Как собрать систему

## Шаг 1. Открой терминал в папке проекта

Перейди в директорию проекта:

```bash
cd /workspace/UTB_Files
```

Если проект у тебя лежит в другом месте, просто перейди в свою папку с этим репозиторием.

## Шаг 2. Очисти старую сборку

Это не обязательно, но полезно:

```bash
make clean
```

Команда удалит старые файлы из папки `build/`.

## Шаг 3. Собери образ

Теперь выполни:

```bash
make
```

Если всё хорошо, появятся файлы:

- `build/boot.bin`
- `build/kernel.bin`
- `build/utb_os.img`

Главный файл здесь — **`build/utb_os.img`**.
Именно его потом запускает QEMU.

---

# Как запустить систему

## Самый простой способ

После сборки выполни:

```bash
make run
```

Эта команда запускает:

```bash
qemu-system-i386 -drive format=raw,file=build/utb_os.img
```

## Если хочешь запустить вручную

Можно и без `make run`:

```bash
qemu-system-i386 -drive format=raw,file=build/utb_os.img
```

---

# Что ты увидишь после запуска

После старта откроется текстовый интерфейс системы.
На экране будут три встроенных окна:

1. `System Shell`
2. `Monitor`
3. `Profile Inspector`

Это не окна Windows и не GUI-библиотека.
Ядро рисует всё само, напрямую через видеопамять.

---

# Управление внутри системы

Клавиши:

- `1` — выбрать `System Shell`
- `2` — выбрать `Monitor`
- `3` — выбрать `Profile Inspector`
- `←` / `→` — переключать активное окно
- `H` — показать или скрыть help-окно
- `R` — заново прочитать BIOS-метрики и перерисовать экран
- `Esc` — перезагрузка

---

# Что делать, если не работает

## Ошибка: `make` не найден

Значит `make` не установлен или не добавлен в PATH.
На Windows проще всего поставить его через MSYS2.

## Ошибка: `nasm` не найден

Значит не установлен assembler.
Установи `nasm` и попробуй снова.

## Ошибка: `qemu-system-i386` не найден

Значит система собралась, но запускать её нечем.
Установи QEMU.

## Команда `make` отработала, но окно не открылось

Проверь, что:

- `build/utb_os.img` действительно появился;
- `qemu-system-i386` установлен;
- ты запускаешь команду в той же папке проекта.

---

# Что именно делает сборка

Команда `make` делает 3 простые вещи:

1. собирает `boot.asm` в `build/boot.bin`;
2. собирает `kernel.asm` в `build/kernel.bin`;
3. склеивает их в один образ `build/utb_os.img`.

То есть логика очень простая:

- сначала собирается загрузчик;
- потом ядро;
- потом из них делается один загрузочный файл.

---

# Что это за встроенные программы

## 1. System Shell

Показывает текстовую информацию о системе и подсказки.

## 2. Monitor

Показывает низкоуровневые данные:

- объём conventional memory;
- номер boot drive;
- video mode и число колонок;
- equipment word;
- индикатор запаса памяти.

## 3. Profile Inspector

Показывает встроенный JSON-фрагмент как пример данных внутри системы.

# Краткая инструкция совсем в двух командах

Если всё уже установлено, то достаточно вот этого:

```bash
make clean
make run
```

Если `make run` не работает из-за отсутствия QEMU, тогда:

```bash
make
```

А потом установи QEMU и снова запусти:

```bash
make run
```
