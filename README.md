# UTB/OS

Это **реально низкоуровневая** база системы, которая грузится как BIOS-совместимый `x86`-образ и не зависит от ОС-хоста во время выполнения.

## Что именно здесь реализовано

- загрузочный сектор на `x86 assembly`;
- вторая стадия ядра, тоже на `x86 assembly`;
- прямой вывод в видеопамять `0xB8000`;
- ввод с клавиатуры через `BIOS int 16h`;
- примитивный интерфейс в текстовом режиме `80x25`;
- встроенные программы:
  - `System Shell`
  - `Monitor`
  - `Profile Inspector`

Это не HTML, не CSS и не приложение поверх Linux/Windows. Это сырой загрузочный образ, который можно поднять как отдельную систему в эмуляторе.

## Архитектура

### Загрузка

1. `boot.asm` стартует из сектора `0` по адресу `0x7C00`.
2. Bootloader читает следующие сектора диска через `BIOS int 13h`.
3. Ядро загружается по адресу `0x8000`.
4. Управление передаётся в ядро.

### Ядро

Ядро:

- переключает систему в VGA text mode интерфейс;
- рисует окна через прямую запись символов в `0xB8000`;
- получает размер conventional memory через `int 12h`;
- принимает клавиши `1`, `2`, `3`, `R`, `Esc` через `int 16h`.

## Программы

### 1. System Shell
Показывает загрузочный лог, список встроенных программ и подсказки по управлению.

### 2. Monitor
Показывает:

- режим вывода;
- модель ввода;
- размер conventional memory;
- номер boot drive.

### 3. Profile Inspector
Показывает встроенный JSON-фрагмент на основе текущего `ProfileTemplate.json`.

## Как собрать

Нужны только локальные инструменты:

- `nasm`
- `make`
- желательно `qemu-system-i386` для запуска

Сборка:

```bash
make
```

После сборки появится образ:

```bash
build/utb_os.img
```

---

## Как запускать

### Вариант 1: QEMU

```bash
qemu-system-i386 -drive format=raw,file=build/utb_os.img
```

### Управление

Внутри системы:

- `1` — выделить `System Shell`
- `2` — выделить `Monitor`
- `3` — выделить `Profile Inspector`
- `R` — перерисовать экран
- `Esc` — перезагрузка через BIOS

---

## Что развивать дальше

Если хочешь, следующий шаг можно сделать уже **ещё ближе к настоящей ОС**:

1. перейти в protected mode;
2. подключить GDT/IDT;
3. повесить IRQ-клавиатуру вместо BIOS polling;
4. сделать простой файловый драйвер;
5. вынести приложения в отдельные модули;
6. перейти от text mode к настоящему pixel framebuffer.
